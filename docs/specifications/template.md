# Specification area

## Status and scope

State the status, owning design issue, independent reviewer, acceptance record,
and explicit non-goals.

## Terms

Define every term and equality notion needed by the clauses.

## Normative clauses

Give every guarantee a stable `REP-<AREA>-NNN` identifier. Each clause states
observable behavior, expected diagnostics, and its applicability boundary.

## Examples and counterexamples

Include concrete cases that distinguish correct behavior from plausible errors.

## Executable evidence

Link semantic-test records, discrimination checks, and explicit coverage gaps.

## Unresolved questions

Link open questions and design blockers. Write `None — reason` only when none
apply.

## Review record

- Reviewer: person or independent agent role
- Outcome: accepted | changes requested | blocked
- Decision: accepted clauses or smallest requested change
- Evidence: issue comment, review artifact, and tests
- Unresolved disagreements: links or `None — reason`
