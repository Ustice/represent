# Prompts

This directory may hold reusable, versioned prompts for specification drafting,
adversarial review, adapter feasibility analysis, and test-quality review.

Prompts should name their required inputs, expected evidence, role, output
shape, and stopping conditions. They must point agents back to repository
sources of truth rather than duplicate evolving specifications. The
[continuity drill](continuity-drill.md) is the first exercised prompt; its
results are preserved under `research/`.
